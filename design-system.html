Design Principles

六條核心原則，指導所有設計決策。AI 修改時須以這些原則為依據。

Core Principles
01
暖色調中性
主色 Hue 20（橙棕），背景用暖米白。避免冷調純白或高飽和色彩，製造溫暖人性感。
02
Typography 主導層次
Serif 標題製造溫度，Sans 內文保持清晰。大小落差明確，用字體語言而非顏色建立視覺層次。
03
細節動態回饋
每個互動（hover / focus / active）都有對應的 transform 或顏色回饋。介面有生命感，而非靜態圖片。
04
主題無縫切換
所有元件透過 CSS 變數切換，不依賴 JS 干預顏色。Light 與 Dark 的情緒都是溫暖的，非冷峻對比。
05
內容呼吸感
容器最大寬 1080px，閱讀寬度 640–720px。版面不塞滿，讓內容和視線都有舒緩的空間。
06
一致的 Hover 語言
卡片 translateY(-4~-6px) + shadow，按鈕 translateY(-1px)，文字連結 underline 展開。統一的動態語彙。

Accessibility

無障礙設計是基本要求，不是加分項。以下規則所有元件均須遵守。

A11y Checklist
F
focus-visible 外框

2px solid hsl(var(--ring))，outline-offset: 2px。style.css 已全域套用，新元件無需再次定義。

T
最小觸控區域

所有互動元素最小高度 / 寬度 44px。Hamburger 44×44px，Theme Toggle 38×38px（加上 padding 仍滿足）。

A
ARIA 標籤

圖示按鈕需 aria-label，Theme Toggle 需 aria-pressed，active 頁面連結需 aria-current="page"。

M
prefers-reduced-motion

style.css 已全域處理：偵測到設定後，所有 reveal 動畫與 transition 自動停用。scroll reveal 內容仍可見。

C
color-scheme 宣告

color-scheme: light dark 讓瀏覽器滾動條與原生表單元件自動跟隨主題，無需額外 CSS。

H
語義化標題層次

每頁只有一個 h1，接著 h2 → h3 依序遞降，不跳級。Section 標籤（小大寫標題）使用 p 或 span，非標題標籤。
